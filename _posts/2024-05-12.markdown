最开始记录 Fly.io 的时候只写了两行命令。  
它能帮你“跑起来”，但还不足以保证“跑得稳”。

## 最小可用两步

```powershell
pwsh -Command "iwr https://fly.io/install.ps1 -useb | iex"
fly auth login
```

这两步只解决了 CLI 安装和认证。

## 稳定上线建议补齐的五步

1. 初始化应用：`fly launch`
2. 核对区域和实例规格：`fly status`
3. 配置密钥：`fly secrets set KEY=VALUE`
4. 查看实时日志：`fly logs`
5. 校验健康检查：`fly checks list`

## 为什么这五步重要

- `fly launch` 决定了应用和配置是否绑定正确
- `secrets` 决定了生产是否能连数据库等外部服务
- `logs` 和 `checks` 决定了故障出现时你能否快速定位

## 快速排障顺序

1. `fly status` 看实例是否启动
2. `fly logs` 看是否启动即崩溃
3. `fly secrets list` 看关键环境变量是否存在
4. `fly checks list` 看健康探针是否连续失败

## 一句话结论

Fly.io 的门槛不高，但“能部署”和“可运维”是两件事。  
把 launch、secrets、logs、checks 补齐，后续稳定性会高很多。
