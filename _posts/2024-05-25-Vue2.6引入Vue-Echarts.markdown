给 Vue 2.6 项目接图表时，很多人第一反应是查 ECharts 配置。  
我当时踩坑后才确认：真正第一关其实是版本兼容。

## 1. 先确认版本关系，再安装依赖

我当时遇到的核心事实是：

- `vue-echarts@6.7.2` 对 Vue 的要求是 `^2.6.12 || ^3.1.1`

这意味着如果项目在 Vue 2.6.x，理论上可行，但依赖搭配必须准确。

## 2. Vue 2 项目常见安装路径

在 Vue 2.x 场景下，通常先装图表依赖：

```shell
npm install echarts vue-echarts
```

如果项目需要 Composition API 相关能力，再补：

```shell
npm i -D @vue/composition-api
```

这里 `-D` 是开发依赖，适合只在构建和开发链路使用的场景。

## 3. 为什么很多人会误判成“图表代码错了”

因为失败现象通常出现在组件层，比如：

- 组件不渲染
- 控制台报导入错误
- 图表容器存在但没有图形

这很容易让人直接去改 option 或 DOM 尺寸。  
但如果依赖版本本身不匹配，前面这些调整通常都治标不治本。

## 4. 我后来固定的排查顺序

每次接图表我会先做这四步：

1. 确认当前 Vue 主版本  
2. 确认 `vue-echarts` 目标版本的兼容范围  
3. 再决定是否需要 `@vue/composition-api`  
4. 最后才进入图表 option 和样式调试

这样可以避免在“依赖层已经错了”的情况下，反复在业务层打转。

## 一句话结论

Vue 2.x 接 Vue-ECharts 时，最该先做的是版本对齐，不是先写图表配置。  
依赖层一旦对齐，后面的渲染问题通常会简单很多。
