很多课程项目最后都停在一个很熟悉的终点：  
抓几份材料，做几张图，写一份结论，看起来完成了，但整个过程几乎不可复用。

我这次做新能源产业链年报分析时，刻意不想再走这条路。

因为如果只把 60 家公司的年报当成“写结论的素材”，你最多得到一次性的分析稿；  
但如果把它们当成**可比较、可复跑、可继续扩展的文本数据库**，那你得到的就不只是这次作业，而是一块以后还能接着长的研究底座。

这就是这篇文章要讲的核心：  
**我不是先有结论，再回头找材料；而是先把材料做成数据库，再允许结论长出来。**

<h2 id="why-build">为什么我要把年报做成数据库，而不是只写分析报告</h2>

新能源行业最容易被误读的地方，在于它经常被当成一个整体来讲。

一说新能源，很多叙事会自动合并成一团：景气、扩产、价格战、出海、技术升级、政策风险，好像大家都在经历同一件事。  
但实际读年报时，你会很快发现不同环节说的根本不是同一种语言。

上游资源公司更容易谈价格压力和盈利收缩；  
光伏链更容易在 `2024` 年把竞争和风险讲得更重；  
风储企业会更频繁强调出海和结构性增长；  
终端和循环环节又更爱讲生态协同、现金流质量和运营稳健性。

如果你只盯财务表，这些差异会被压平成“几条营收和利润曲线”。  
而年报文本恰好提供了另一层观察入口：企业不只在披露结果，也在披露它认为值得强调、值得解释、值得防御的部分。

所以对我来说，问题就不再是“我要不要读年报”，而是：

1. 能不能把年报中的制度化语言稳定抽出来；
2. 能不能把这些语言和财务变量放进同一个比较框架；
3. 能不能把这套流程沉淀成别人也能复现的研究仓库。

这三件事做成了，结论才不是一次性的。

<h2 id="pipeline">这套研究底座到底是怎么搭起来的</h2>

这套仓库从一开始就不是按“作业文件夹”来组织的，而是按更接近公开研究仓库的方式组织的。

项目入口定义，写在仓库的 `README.md` 里。

它的核心问题是：

> 新能源产业链上市公司年报文本，能否系统反映行业景气分化、经营压力与风险叙事？

围绕这个问题，整个流程被拆成四层。

### 1. PDF 输入层

最上游是 CNINFO 年报 PDF。

这里最重要的不是“下载到文件”，而是先把样本口径定清楚：

- 样本公司：60 家新能源产业链上市公司
- 主样本：`2022-2024`
- 补充样本：`2025` 已披露年报
- 数据来源：CNINFO 年报 PDF

这个口径看起来像学术写法，但实际上是工程前提。  
只要样本边界不清，后面所有变量、图表和比较都会漂。

### 2. 章节抽取层

原始 PDF 不能直接拿来做研究比较，所以第二步不是全文乱扫，而是先抽核心章节：

- `MD&A`
- `Risk Factors`
- `Principal Business`
- `Financial Highlights`

这一步很关键，因为它决定你后面得到的文本变量到底在说什么。

如果不先按披露结构做锚定，你很容易把：

- 例行法务语言
- 页脚注释
- 报表格式碎片
- 与经营无关的模板话术

全部混进来，最后做出一堆看似精确、实际噪音很高的词频。

### 3. 结构化变量层

第三层才是把年报真正变成“可以比较的数据”。

这一层同时做两类变量：

一类是财务变量：

- 营业收入
- 归母净利润
- 经营现金流
- 总资产
- 净资产
- 收入同比、利润同比等派生变量

另一类是文本变量：

- 价格压力
- 扩产
- 出海
- 技术升级
- 风险
- 竞争
- 生态协同

这一步对应的核心数据产物包括：

- `master_table.csv`
- `financial_metrics.csv`
- `text_features.csv`
- `report_sections.csv`

数据库的第一版到这里，其实已经足够支持课程级的图表和描述性分析。  
但如果只停在这里，研究仍然有一个明显上限：**章节级粒度太粗。**

<h2 id="v2-upgrade">为什么我又把它升级成了 V2 数据库</h2>

我后来继续往下做时，很快发现一个问题：

章节级文本变量能告诉你“这一章里风险词变多了”，  
但它很难告诉你：

- 风险具体发生在什么段落；
- 它更接近价格压力，还是海外扩张受阻；
- 里面有没有明确的时间线索和数值线索；
- 不同公司是不是在用相似但不完全相同的话讲同一种经营事件。

这就是为什么我后来又做了 `V2`。

V2 的升级说明，写在仓库里的 `database_v2_bilingual.md` 文档中。

V2 的升级方向很明确：

1. 从章节级升级到段落级
2. 从简单词频升级到事件候选级
3. 从静态文本升级到时间/数值线索级
4. 为后续语义层和聚类分析留出口

目前 V2 里最关键的几张表是：

- `paragraph_table_v2.csv`
- `event_candidate_table_v2.csv`
- `time_value_table_v2.csv`
- `company_year_summary_v2.csv`

这套结构让数据库开始更像“研究基础设施”，而不是“课程交付附件”。

因为它支持的已经不只是“平均词频比较”，还包括：

- 事件候选展开
- 项目时间线整理
- 数值线索抽取
- 公司间叙事相似性比较
- 后续 embedding 和 cluster 分析

V2 的一个核心判断我非常认同：**规则层不能被当成终点。**

规则适合做：

- 章节锚定
- 时间和数值表达抽取
- 高召回事件候选

但规则不适合做：

- 复杂语义判断
- 同义表达泛化
- 跨公司叙事聚类

所以 V2 的方向不是“抛弃规则”，而是“规则 + 语义”的组合结构。

<h2 id="findings">这套数据库最后让我看到了什么</h2>

如果只问最后的研究发现，核心结论其实不复杂：

**新能源产业链并不是同步变化的，它在文本层面表现出非常明显的景气分化。**

具体来看：

### 1. 光伏链在 `2024` 的价格压力和风险叙事更重

这部分是最直观的。

年报文本里，光伏链公司更容易同时出现：

- 价格竞争
- 风险抬升
- 经营承压

如果你只看产业新闻，这件事容易变成一句“大环境不好”；  
但放回制度化披露文本里，它会变成可以系统比较的信号。

### 2. 上游资源企业没有彻底停止扩张叙事，但价格压力和盈利收缩信号在加强

这类公司不是简单从“扩张”跳到“收缩”。  
更真实的状态是两类语言并存：

- 一边还在谈扩产与资源布局
- 一边又不得不谈价格压力与盈利变化

这种语言并存，本身就是行业转折期的重要特征。

### 3. 风储链更强调出海和结构性增长

风电和储能环节的文本更容易释放一种“增长叙事还没有完全坍塌”的感觉。

它们更频繁地讨论：

- 海外市场
- 结构性需求
- 中长期扩张空间

这和光伏链的“竞争压力前置化”形成了很明显的对照。

### 4. 终端与循环环节更看重生态协同与现金流质量

这类公司更像是在强调“怎么活得稳”，而不是“怎么讲得快”。

它们的年报里，更容易出现：

- 协同
- 生态
- 现金流质量
- 运营稳健性

这说明同样身处新能源链条，不同环节在同一时点最关心的问题并不一样。

而这恰恰是文本数据库比财务表更有价值的地方：  
它能把“企业正在优先防御什么、解释什么、押注什么”显影出来。

<h2 id="value">这类项目真正值钱的地方在哪里</h2>

如果把这件事只理解成“写一份行业分析报告”，它的价值很快就到头了。  
因为报告一交，流程也就结束了。

但如果把它理解成“搭一套研究底座”，价值就完全不同。

我认为它真正值钱的地方有三层。

### 第一层：它把制度化披露文本变成了可比较的数据对象

这件事并不天然成立。

年报当然一直在那里，但如果没有：

- 样本口径
- 章节锚点
- 变量定义
- 质量报告
- 数据表结构

它仍然只是一堆文件，不是一套研究对象。

### 第二层：它让“课程作业”往“公开研究仓库”走近了一步

这一点在仓库结构里特别明显。

你能看到的不只是交付物，还有：

- `src/` 可复用代码
- `configs/` 规则与关键词
- `docs/` 方法与说明
- `tests/` 测试
- `outputs/` 质量报告和图表

这说明目标不是“交完就算”，而是至少在组织方式上，开始按可复现项目来要求自己。

### 第三层：它给后续更强的研究方法留了接口

V2 的意义就在这里。

它不是为了显得复杂，而是为了让后面的东西有落点：

- 段落级相似性检索
- 事件抽取
- 时间线建模
- 语义聚类
- 行业叙事漂移分析

如果前面的数据库结构不对，后面这些都接不上。

<h2 id="next-step">还差什么，下一步该怎么继续做</h2>

这套系统当然还没到“可以发论文”的程度，缺口也很明确。

第一，PDF 结构化能力还可以更强。  
后面如果要提升精度，应该引入更成熟的 PDF 结构化引擎，而不是只依赖当前流程。

第二，规则层还需要继续细化。  
尤其是一些行业特定表达、研发投入、项目进展和风险表述，单靠当前字典还不够。

第三，语义层值得继续做深。  
现在 V2 已经给 embedding 和 clustering 留了口，但后面如果真想做跨公司叙事比较，还需要更细的语义分类和人工校验。

第四，研究叙事还可以进一步前移到“问题驱动”。  
现在这套数据库已经足够支撑：

- 哪些环节的价格压力最先在文本里抬头
- 哪些环节更爱讲出海
- 哪些公司在景气回落期更强调现金流

这些都可以成为下一轮更聚焦的研究问题。

如果一句话总结这篇文章：

**我做的不是“新能源行业读后感”，而是一块把制度化披露文本转成可复现研究对象的底座。真正重要的，不是某一页图表，而是这套底座以后还能继续长。**
