学 Docker 最容易学偏成“命令背诵”。  
真正长期有用的是环境思维：依赖、启动顺序、配置和数据边界能否被明确描述。

## 起步先问四个问题

1. 应用依赖哪些系统组件
2. 哪些依赖应该写进镜像
3. 哪些配置必须走环境变量
4. 数据目录是否与容器生命周期解耦

## 三条基础命令够你排一半问题

```bash
docker ps
docker logs <container_id>
docker exec -it <container_id> sh
```

- `ps` 看容器状态
- `logs` 看进程报错
- `exec` 进容器核对真实运行环境

## 一句话结论

Docker 不是“再学一套命令”，而是把运行环境从隐形约定变成显式资产。
