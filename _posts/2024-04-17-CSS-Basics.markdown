我最初只留了一个链接：

- [MDN CSS basics](https://developer.mozilla.org/en-US/docs/Learn/Getting_started_with_the_web/CSS_basics)

后来回看，这个问题其实很典型：  
CSS 很多人“学过很多次”，但真正做页面时还是会乱。

## 为什么会反复卡住

因为 CSS 最容易学成“属性记忆”，而不是“布局能力”。

你知道 `margin`、`padding`、`position`，不代表你能稳定搭出：

- 对齐稳定的卡片
- 可伸缩的导航
- 不打架的层级
- 移动端不崩的布局

## 我后来更推荐的练法

先做四组小练习：

1. 盒模型和间距系统  
2. Flex + Grid 双路径布局  
3. 定位与层级（`position` + `z-index`）  
4. 响应式断点与字体缩放

每组只做一个小页面，但必须做到“从桌面到移动都不崩”。

## 一句话结论

CSS 基础不怕少，怕散。  
把练习从“记属性”改成“做布局”，学习曲线会明显稳定。
